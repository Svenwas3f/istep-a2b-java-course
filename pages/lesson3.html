Part 1: What Are Classes and Objects?

Understanding Objects in the Real World

Think about your school. Every student in your school is different - they have different names, ages, interests, and grades. But all students have something in common:

  • All students have a name
  • All students have an age
  • All students can study
  • All students can eat lunch
  • All students can take tests

In Java programming:

  • A Class is like a blueprint or template. It describes what a student is and what a student can do (the general plan)
  • An Object is a specific instance based on that blueprint (like your friend Alex, who is a specific student)

Real-world analogy:

  • A Class is like a cookie cutter - it defines the shape
  • Objects are the actual cookies - they all have the same shape, but each one is unique

Another Example: Cars

Imagine all cars in the world:

  • The Class "Car" defines: color, brand, speed, fuel level, doors
  • The Class "Car" defines actions: accelerate, brake, turn left, honk
  • Your Object (a specific car): a red Toyota with 4 doors is one specific instance
  • My Object (another car): a blue Honda with 2 doors is another specific instance

Classes Contain Two Things:

1. Attributes (Properties/Data)

What the object looks like and what it stores:

  • For a Student class: name, age, grade, studentID
  • For a Car class: color, brand, speed, fuel
  • For a Pet class: name, age, type (dog/cat/bird)

2. Methods (Actions/Behaviors)

What the object can do:

  • For a Student class: study(), takeTes(), eatLunch()
  • For a Car class: accelerate(), brake(), honk()
  • For a Pet class: bark(), eat(), sleep()

Part 2: Creating Your First Class

Now let's write actual code! We'll create a simple Student class.

Step 1: Create the Class File

  1. Create a new folder called lesson3 inside your javacourse package
  2. Create a new file called Student.java

Step 2: Write the Student Class

package al.istep.javacourse.lesson3;

public class Student {
    // Attributes (Properties)
    public String name;
    public int age;
    public String grade;
    public double gpa;
    
    // Method: Study
    public void study(String subject) {
        System.out.println(name + " is studying " + subject);
    }
    
    // Method: Take a test
    public void takeTest(String testName) {
        System.out.println(name + " is taking the " + testName + " test");
    }
    
    // Method: Eat lunch
    public void eatLunch() {
        System.out.println(name + " is eating lunch");
    }
    
    // Method: Print student info
    public void printInfo() {
        System.out.println("Student: " + name);
        System.out.println("Age: " + age);
        System.out.println("Grade: " + grade);
        System.out.println("GPA: " + gpa);
    }
}

Let's understand what we created:

Attributes (Lines 5-8):

public String name;
public int age;
public String grade;
public double gpa;

These are variables that every Student object will have. They store information about each student.

Methods (Lines 10-26):

Each method is an action a student can perform:

  • study(String subject) - The student studies a specific subject
  • takeTest(String testName) - The student takes a named test
  • eatLunch() - The student eats lunch (no extra information needed)
  • printInfo() - Displays all the student's information

Step 3: Create a Program to Use Your Class

Now let's create objects (instances) from our Student class!

package al.istep.javacourse.lesson3;

Understanding Object Creation

Let's break down this line:

Student student1 = new Student();
  • Student - This is the class type (like int, double, String)
  • student1 - This is the variable name (the object)
  • new - This keyword creates a new object in memory
  • Student() - This calls the constructor (creates the object)

Then we fill in the attributes:

student1.name = "Alex";
student1.age = 14;

The dot . is used to access attributes and methods of an object.

Part 3: Introduction to the String Class

You've actually been using a class all along! The String class is one of the most important built-in classes in Java.

What is a String?

A String is a sequence of characters (text). It's actually an Object created from the String class!

String message = "Hello, World!";
String name = "Alex";

Even though we've been treating strings like simple variables, they're actually objects with built-in methods and properties!

Common String Methods

1. Length - Finding the number of characters

package al.istep.javacourse.lesson3;

3. charAt() - Getting a character at a specific position

package al.istep.javacourse.lesson3;

public class StringExample3 {
    public static void main(String[] args) {
        String word = "Java";
        
        // Position 0 = 'J', Position 1 = 'a', Position 2 = 'v', Position 3 = 'a'
        System.out.println("Character at position 0: " + word.charAt(0));
        System.out.println("Character at position 1: " + word.charAt(1));
        System.out.println("Character at position 2: " + word.charAt(2));
        System.out.println("Character at position 3: " + word.charAt(3));
    }
}

Output:

Character at position 0: J
Character at position 1: a
Character at position 2: v
Character at position 3: a

Important: Remember that counting starts at 0, not 1!

4. substring() - Extract part of a string

World

5. indexOf() - Finding where text appears

package al.istep.javacourse.lesson3;

public class StringExample5 {
    public static void main(String[] args) {
        String sentence = "The quick brown fox";
        
        int index1 = sentence.indexOf("quick");
        int index2 = sentence.indexOf("fox");
        int index3 = sentence.indexOf("z");  // Not found
        
        System.out.println("Index of 'quick': " + index1);  // 4
        System.out.println("Index of 'fox': " + index2);    // 16
        System.out.println("Index of 'z': " + index3);      // -1 (not found)
    }
}

Output:

Index of 'quick': 4
Index of 'fox': 16
Index of 'z': -1

Note: If the text is not found, indexOf() returns -1

Part 4: Creating More Complex Classes

Example 1: Car Class

package al.istep.javacourse.lesson3;

public class Car {
    // Attributes
    public String brand;
    public String model;
    public String color;
    public int year;
    public double speed;
    public double fuelLevel;
    
    // Methods
    public void accelerate(double amount) {
        speed += amount;
        fuelLevel -= (amount * 0.5);
        System.out.println(brand + " " + model + " accelerates. Speed: " + speed + " km/h");
    }
    
    public void brake() {
        speed = 0;
        System.out.println(brand + " " + model + " stops. Speed: 0 km/h");
    }
    
    public void honk() {
        System.out.println(brand + " " + model + " says: HONK HONK!");
    }
    
    public void displayInfo() {
        System.out.println("Car: " + year + " " + brand + " " + model);
        System.out.println("Color: " + color);
        System.out.println("Current Speed: " + speed + " km/h");
        System.out.println("Fuel Level: " + fuelLevel + "%");
    }
}

Part 5: Important Concepts

1. The public Keyword

When we write public String name;, we're saying that anyone can access this attribute from outside the class. The public keyword makes it accessible.

2. The this Keyword (Advanced)

Inside a class, you can use this to refer to the current object:

public void printName() {
    System.out.println(this.name);  // Same as just: System.out.println(name);
}

3. Creating Objects vs. Using Classes

  • The class is the blueprint (Student.java)
  • The object is an instance of that class (student1, student2)
  • You can create many objects from one class

Common Mistakes and How to Fix Them

Mistake 1: Forgetting the new keyword

Wrong:

Student student1;
student1.name = "Alex";  // ‚ùå Error: student1 is null

Correct:

Student student1 = new Student();  // ‚úÖ Create the object first
student1.name = "Alex";

Mistake 2: Using method name with parentheses when accessing attributes

Wrong:

int length = message.length();  // ‚úÖ This IS correct - length() is a method
String upper = text.toUpperCase;  // ‚ùå Missing parentheses

Correct:

int length = message.length();           // ‚úÖ Method call
String upper = text.toUpperCase();       // ‚úÖ Method call with ()

Mistake 3: String comparison with ==

Wrong:

String password1 = "secret";
String password2 = "secret";
if (password1 == password2) {  // ‚ùå Wrong way to compare
    System.out.println("Same!");
}

Correct:

String password1 = "secret";
String password2 = "secret";
if (password1.equals(password2)) {  // ‚úÖ Right way to compare strings
    System.out.println("Same!");
}
